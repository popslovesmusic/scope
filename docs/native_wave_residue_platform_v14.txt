2. ARCHITECTURAL BREAKTHROUGHS (PHASE 1)
----------------------------------------

A. Operator-Selected Local Frames (Patch 18/19)
   - Problem: Continuous phase flow was often disrupted by "viewing angle" 
     jumps in the raw hypersphere projection.
   - Solution: Implemented automatic operator selection (select_operator) 
     that finds the optimal reference frame (++, --, +-, -+) to minimize 
     apparent mismatch between frames.
   - Result: Seamless phase continuity regardless of global orientation shifts.

B. Active Trace Feedback (Patch 20)
   - Problem: Residue memory was long-term/static, leaving the short-term 
     continuation loop vulnerable to noise.
   - Solution: Introduced 'trace_segments'—a buffer of recent aligned velocities 
     (phi_t - phi_{t-1}). This serves as an active directional bias 
     (trace_feedback_vector) added to the local momentum.
   - Result: Internal "grooves" that actively pull the trajectory toward 
     previously successful paths.

C. Persistent Success Evaluation (Patch 20)
   - The platform now evaluates each traversal pass. If the mismatch trend 
     improves (last_mean <= first_mean), the 'successful_traversals' count 
     increments in persistent memory.
   - The feedback gain (beta) scales exponentially with these successes, 
     making well-traveled paths increasingly "low-friction."

D. Selective Groove Decay (Patch 19)
   - Grooves are no longer global averages. On divergence (mismatch > 0.02), 
     the trace buffer is aggressively pruned to preserve only recent context.
   - This prevents "smearing" of the groove across different signal types.
